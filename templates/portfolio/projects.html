1).REAL-TIME API DATA VISUALISATION.

The concept of working efficiently with the API of a particular technology is what this project demonstrates. This includes the following steps:- accessing the API and making an API call via Python's request object, verifying and translating the data returned, and using Python's Plotly module to analyse and create a visualisation of this data.
The result is a neat graph with active links showing real-time data at that moment. Below is one such graph which shows real-time data in the form of "top news headlines in the USA". Each time the code is run, the graph updates to showthe most current news articles. I was able to accomplish this by working with the NEWS API and by using a unique API-key. Below is the resulting visualisation. My code can quickly be accessed in my GitHub repository.
There is a similar visualisation to this one, drawn alongside. It was generated using data from HACKER NEWS API. At any instance of running code, it will show the most commented topics in Hacker News at that moment. 2). A BOOK-MARKING WEB APPLICATION.

This entry features a book-marking django web application which allows users the ability to like 'Categories' and add 'Pages' to specific select categories. Python is used alongside JavaScript, JQuery, and AJAX in order to implement and simulate dynamic features.
Authenticated users can freely interact with all features of the site which include but not limited to:- profile accounts with enabled profile-image uploads, personal website entries, viewing of other user profiles, and bio-message displays.
Styling was accomplished by custom CSS and Bootstrap4 application. Testing and debugging was done prior to deployment of the application online.
The application is fully-functional, readily accessible online, and goes by the name "Page-House". The copyright seal is also denoted on the 'About Page' of the app.

3). A PYGAME WITH PYTHON.

This is a game development project comprising of a 2-D game built using Pygame, a Python module. The game was created and designed using Python's native class objects, and by consistently splitting code into manageable potions for further development. Writing this game offered me an opportunity to enhance my game development and problem-solving skills. I had to grapple with the many challenges I faced, climb over huddles to create a working, playable game. I have titled this game as 'Game Character'; a particular game object (the player) combats enemy ships moving in a systematic manner up, down, and across the screen. The game features escalating levels, a scoring system, an updated high score and limited player-lives. I enhanced it by introducing sound and music effects using Pygame Mixer module.

I have included a snapshot of my program and a short video file below to demonstrate my work. The complete code for this game can be accessed in my GitHub repository to which I am linking here.

5). VISUALISING GLOBAL DATASETS.

This section depicts striking visualisations of recent global earthquakes in the scale of 1.0 and above, generated using Plotly. The methodology used involved analysing geo-json earthquake data using Python's objects and modules.
Plotly's Graph Object enabled useful access to Scattergeo and Layout which were used to generate the world map. The relevant data to be plotted was obtained by extracting relevant features from json data, and appending them into Python-lists. The output is as shown below.

Hovering the cursor over the colored dots on the map, allows a label to pop-up highlighting relevant information in a useful way. The size of the dots correspond to the earthquake magnitude in that location.
The color-scale used here, "Viridus", was deliberately chosen in order to highlight the map in a theme suited to earthquakes. The background code for this work can be found in this repository.
